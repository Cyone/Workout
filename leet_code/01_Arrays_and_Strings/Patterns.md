# Problem Solving Patterns: Arrays and Strings

## Common Patterns
1. **Two Pointers**
2. **Sliding Window**
3. **Prefix Sum**
4. **Fast & Slow Pointers**

## How to Detect These Patterns

### 1. Two Pointers
- **When to use**: You need to search for a set of elements (pairs, triplets, etc.) that satisfy a certain condition, usually in a sorted array.
- **Keywords**: "sorted array", "pair that sums to target", "remove duplicates in-place", "reverse".
- **Example**: Two Sum II, 3Sum, Valid Palindrome, Container With Most Water.

### 2. Sliding Window
- **When to use**: You need to find a subarray, substring, or a contiguous sequence that satisfies a specific condition (e.g., longest, shortest, max sum).
- **Keywords**: "contiguous subarray", "longest substring", "minimum window", "k consecutive elements".
- **Example**: Longest Substring Without Repeating Characters, Maximum Average Subarray I, Minimum Size Subarray Sum.

### 3. Prefix Sum
- **When to use**: You need to frequently query the sum of elements in a range or subarray.
- **Keywords**: "sum of subarray", "range query", "cumulative sum", "equilibrium index".
- **Example**: Range Sum Query - Immutable, Subarray Sum Equals K.

### 4. Fast & Slow Pointers
- **When to use**: Cycle detection or finding the midpoint (often on arrays representing cycles/linked lists).
- **Keywords**: "cycle", "loop", "find the duplicate number".
- **Example**: Find the Duplicate Number, Linked List Cycle (applied to array via indices).

# Problem Solving Patterns: Linked Lists

## Common Patterns
1. **Fast & Slow Pointers (Tortoise and Hare)**
2. **In-place Reversal of a LinkedList**
3. **Dummy Node / Sentinel Node**
4. **Two Pointers (Standard)**

## How to Detect These Patterns

### 1. Fast & Slow Pointers
- **When to use**: You need to detect a cycle, find the middle of the linked list, or find the k-th node from the end.
- **Keywords**: "cycle", "loop", "middle node", "k-th from the end".
- **Example**: Linked List Cycle, Middle of the Linked List, Palindrome Linked List.

### 2. In-place Reversal of a LinkedList
- **When to use**: You are asked to reverse the links between nodes of a linked list without using extra space.
- **Keywords**: "reverse", "in-place", "reverse sub-list", "reverse in k-groups".
- **Example**: Reverse Linked List, Reverse Linked List II, Reverse Nodes in k-Group.

### 3. Dummy Node / Sentinel Node
- **When to use**: The head of the linked list might be removed, or you are merging lists and need a generic starting point.
- **Keywords**: "remove elements", "merge lists", "delete head".
- **Example**: Remove Nth Node From End of List, Merge Two Sorted Lists, Remove Linked List Elements.

### 4. Two Pointers (Standard)
- **When to use**: Traversing two different lists simultaneously or keeping a gap between two pointers.
- **Keywords**: "intersection", "merge", "k-th from end".
- **Example**: Intersection of Two Linked Lists.

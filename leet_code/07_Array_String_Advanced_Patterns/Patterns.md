# Problem Solving Patterns: Array/String Advanced

## Common Patterns
1. **Top 'K' Elements (Heaps)**
2. **K-way Merge**
3. **Greedy Algorithms**
4. **Dynamic Programming (1D/2D)**

## How to Detect These Patterns

### 1. Top 'K' Elements (Heaps)
- **When to use**: You are asked to find the top, smallest, or most frequent 'K' elements among a given set.
- **Keywords**: "top K", "Kth largest", "Kth smallest", "frequent".
- **Example**: Kth Largest Element in an Array, Top K Frequent Elements, Merge K Sorted Lists (can also be K-way merge).

### 2. K-way Merge
- **When to use**: You are given 'K' sorted arrays or lists and need to merge them into a single sorted list.
- **Keywords**: "merge k sorted", "kth smallest in m sorted arrays".
- **Example**: Merge K Sorted Lists, Kth Smallest Element in a Sorted Matrix.

### 3. Greedy Algorithms
- **When to use**: You need to make a locally optimal choice at each step with the hope of finding a global optimum. Often used for optimization problems.
- **Keywords**: "maximum profit", "minimum jumps", "gas station", "non-overlapping".
- **Example**: Jump Game, Gas Station, Task Scheduler, Non-overlapping Intervals.

### 4. Dynamic Programming
- **When to use**: overlapping subproblems and optimal substructure. 
- **Keywords**: "maximum subarray", "longest palindromic substring".
- **Example**: Kadane's Algorithm, Longest Palindromic Substring.

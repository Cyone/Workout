# Problem Solving Patterns: Essential Algorithms

## Common Patterns
1. **Binary Search**
2. **Merge Intervals**
3. **Cyclic Sort**
4. **Subsets / Backtracking**

## How to Detect These Patterns

### 1. Binary Search
- **When to use**: You are looking for an element (or a value satisfying a condition) in a sorted array or a monotonic function domain.
- **Keywords**: "sorted array", "O(log N) time", "search pivot", "find minimum in rotated array".
- **Example**: Search in Rotated Sorted Array, Find Minimum in Rotated Sorted Array, Koko Eating Bananas.

### 2. Merge Intervals
- **When to use**: You are dealing with overlapping intervals and need to merge them, find intersections, or find free time.
- **Keywords**: "intervals", "overlapping", "merge", "meeting rooms".
- **Example**: Merge Intervals, Insert Interval, Meeting Rooms II.

### 3. Cyclic Sort
- **When to use**: You are dealing with an array containing numbers in a given range (e.g., 1 to N, or 0 to N) and asked to find missing or duplicate numbers.
- **Keywords**: "numbers from 1 to N", "missing number", "duplicate number", "in-place O(N)".
- **Example**: Missing Number, Find All Duplicates in an Array, First Missing Positive.

### 4. Subsets / Backtracking
- **When to use**: You need to generate all possible combinations, permutations, or subsets of a given set.
- **Keywords**: "all combinations", "permutations", "subsets", "generate parentheses", "n-queens".
- **Example**: Subsets, Permutations, Combination Sum, Generate Parentheses.

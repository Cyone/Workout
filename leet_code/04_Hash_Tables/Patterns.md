# Problem Solving Patterns: Hash Tables (Maps & Sets)

## Common Patterns
1. **Frequency Counting**
2. **Two Sum Pattern (Complement Search)**
3. **Grouping or Categorizing**
4. **Sliding Window with Hash Map**

## How to Detect These Patterns

### 1. Frequency Counting
- **When to use**: You need to count the occurrences of elements or check if an element appears a certain number of times.
- **Keywords**: "majority element", "anagram", "most frequent", "unique elements".
- **Example**: Valid Anagram, Majority Element, First Unique Character in a String.

### 2. Two Sum Pattern (Complement Search)
- **When to use**: You are looking for a pair of numbers that satisfy a condition (like A + B = Target), and order doesn't dictate a two-pointer approach.
- **Keywords**: "two sum", "difference is k", "pair".
- **Example**: Two Sum, Subarray Sum Equals K.

### 3. Grouping or Categorizing
- **When to use**: You need to group elements by a specific characteristic or derived key.
- **Keywords**: "group anagrams", "group by pattern", "isomorphic".
- **Example**: Group Anagrams, Isomorphic Strings, Word Pattern.

### 4. Sliding Window with Hash Map
- **When to use**: Finding the longest/shortest substring that contains a certain number of distinct characters or all characters of another string.
- **Keywords**: "longest substring with K distinct characters", "minimum window substring", "anagrams in a string".
- **Example**: Minimum Window Substring, Find All Anagrams in a String, Longest Substring Without Repeating Characters.
